# Architecture

> Generated by /map on 2026-04-30

## Overview

ZENO (Just A Rather Very Intelligent System) is a **voice-first, locally-running personal AI assistant** for developers and knowledge workers. It runs as a native desktop application (Tauri shell) with a Python backend daemon and a Rust/Tauri frontend UI shell.

ZENO listens for a configurable wake word ("Hey Zeno"), transcribes speech via OpenAI Whisper, classifies intent using a rule-based NLP parser (falling back to the Claude API for ambiguous or complex language tasks), and dispatches structured actions against a local SQLite database. Passive monitoring (app activity, browser tabs) feeds an analytics pipeline that generates morning briefings and focus insights.

The system is **privacy-first**: all data stays local. The only external calls are to the Claude API (for reasoning/generation) and optionally to ElevenLabs (for premium TTS).

## Components

### Voice Pipeline
- **Purpose:** Listens for wake word, transcribes speech, passes raw transcript to NLP parser
- **Location:** `~/Zeno/voice/` (Python module — not yet scaffolded)
- **Technology:** OpenAI Whisper (model: `whisper-base` configurable), sounddevice or pyaudio for mic capture
- **Activation modes:**
  - `"Hey Zeno"` — primary wake word (10s timeout)
  - `"Zeno, <command>"` — session mode (no timeout)
  - `Ctrl+Shift+Space` — brain dump hotkey overlay
  - `Ctrl+Shift+J` (held) — push-to-talk fallback
- **Output:** Raw transcript string → NLP Intent Parser

### NLP Intent Parser
- **Purpose:** Classifies voice utterances into structured intent objects (10 categories, 52 named intents)
- **Location:** `~/Zeno/nlp/` (Python module — not yet scaffolded)
- **Intent Object Schema:**
  ```json
  {
    "intent_category": "task_management",
    "intent_name": "add_task",
    "slots": { "title": "...", "due_date": "...", "priority": "..." },
    "confidence": 0.94,
    "raw_transcript": "..."
  }
  ```
- **Confidence threshold:** 0.75 — below this, clarification dialog is triggered
- **31 slot types** spanning temporal (date, duration, range), entity (task_ref, project_ref, app_name), and scalar/enum (priority, mood, block_type) categories
- **Multi-intent splitting:** conjunctions trigger multiple intent objects
- **Dependents:** Action Dispatcher

### Claude API Engine
- **Purpose:** Powers reasoning-heavy tasks: morning briefings, rubber duck PRD generation, weekly insight narratives, slot inference, clarification disambiguation
- **Location:** `~/Zeno/ai/` (Python module — not yet scaffolded)
- **Model:** `claude-sonnet-4` (configurable in `user_profile`)
- **Context sourced from:** Session files, SQLite queries, behaviour_patterns
- **Output:** Markdown text, task lists, PRD documents, JSON
- **Dependents:** Action Dispatcher, Scheduler, Rubber Duck Mode

### Action Dispatcher
- **Purpose:** Routes resolved intent objects to the correct handler function
- **Location:** `~/Zeno/dispatcher/` (Python module — not yet scaffolded)
- **Key flows dispatched:**
  - `task_*` intents → SQLite `tasks` table CRUD
  - `session_*` intents → SQLite `sessions` table + file writes
  - `workspace_*` intents → Macro Engine
  - `reminder_*` intents → APScheduler jobs + SQLite `reminders`
  - `analytics_*` intents → SQLite views / analytics tables
  - `rubber_duck_*` intents → Claude API Engine + file writes
  - `notes_*` intents → SQLite `notes` + FTS5
- **Dependents:** Macro Engine, TTS Engine, SQLite, Scheduler

### SQLite Database (`Zeno.db`)
- **Purpose:** Single source of truth for all persistent state
- **Location:** `~/Zeno/Zeno.db`
- **Mode:** WAL (Write-Ahead Logging), foreign keys ON, 256MB mmap
- **Schema version:** v1.0 (720-line schema)
- **14 sections:**

| Section | Tables | Purpose |
|---------|--------|---------|
| 1 — Core Identity | `user_profile`, `privacy_exclusions` | Singleton config, privacy blocklist |
| 2 — Projects | `projects` | Project registry with status, color, paths |
| 3 — Tasks | `tasks`, `task_dependencies`, `task_history` | Full task lifecycle + audit trail |
| 4 — Sessions | `sessions`, `session_pending_items` | Daily work sessions with memory state |
| 5 — Time Blocks | `time_blocks`, `schedule_reroutings` | Calendar scheduling + rerouting log |
| 6 — Reminders | `reminders` | One-time, recurring, contextual reminders |
| 7 — Activity | `activity_log`, `browser_sessions`, `context_switches` | 30-sec passive monitoring samples |
| 8 — Analytics | `analytics_weekly`, `analytics_daily`, `behaviour_patterns` | Computed stats + ML-lite patterns |
| 9 — Workspaces | `workspaces`, `workspace_steps`, `workspace_activations` | Macro profiles + execution steps |
| 10 — Voice | `voice_interactions`, `conversations` | Full NLP interaction audit |
| 11 — Notes | `notes` (+ FTS5 virtual table) | Searchable brain dumps |
| 12 — Rubber Duck | `rubber_duck_sessions` | Socratic planning sessions |
| 13 — App Class | `app_classifications` | Work/off-task whitelist |
| 14 — System | `system_events`, `schema_version` | Audit log + migrations |

- **4 convenience views:** `v_todays_tasks`, `v_todays_schedule`, `v_morning_brief_tasks`, `v_weekly_activity`
- **Auto-maintenance triggers:** `updated_at` timestamps, task status history log, browser dwell calculation
- **FTS5:** Full-text search on `notes` table (title, content, tags)

### Macro Engine
- **Purpose:** Executes workspace setup sequences — launches apps, opens URLs, arranges windows, toggles DND
- **Location:** `~/Zeno/macros/` (Python module — not yet scaffolded)
- **Safety:** All actions execute from `app_classifications` whitelist; no arbitrary shell commands
- **Step types:** `open_app`, `open_url`, `focus_window`, `arrange_windows`, `toggle_dnd`, `announce`, `wait_ms`
- **Dependents:** Action Dispatcher (via workspace intents)

### TTS Engine
- **Purpose:** Converts Zeno's text responses to spoken audio output
- **Location:** `~/Zeno/tts/` (Python module — not yet scaffolded)
- **Supported engines:** `pyttsx3` (offline default), `elevenlabs` (premium, API), `coqui` (offline alternative)
- **Configurable:** per `user_profile.tts_engine`
- **Dependents:** Action Dispatcher (response_text field)

### Activity Monitor
- **Purpose:** Passively samples the active window every 30 seconds; infers project/task context; detects off-task behavior
- **Location:** `~/Zeno/monitor/` (Python module — not yet scaffolded)
- **Sampling:** 30-second intervals → writes to `activity_log`
- **Privacy:** Window titles are redacted per `privacy_exclusions` rules before storage
- **Metrics captured:** `app_name`, `window_title`, `input_level` (idle/light/moderate/heavy), `wpm_bucket`, `is_off_task`
- **Dependents:** Analytics pipeline

### Browser Extension
- **Purpose:** Tracks active browser tabs, domain dwell time, and URL categories via WebSocket to the Python daemon
- **Location:** `~/Zeno/extension/` (browser extension — not yet scaffolded)
- **Supported browsers:** Chrome, Firefox, Edge, Safari
- **Privacy:** Domains redacted per `privacy_exclusions` before storage
- **Output:** → `browser_sessions` table

### Tauri UI Shell
- **Purpose:** Provides the desktop UI: system tray icon, brain dump overlay (non-focus-stealing), analytics dashboard, daily schedule view, settings
- **Location:** Frontend (Rust/Tauri + likely React/TS — not yet scaffolded)
- **Key views:**
  - System tray quick capture
  - Hotkey-triggered brain dump overlay
  - Analytics dashboard (deep work charts, off-task breakdown)
  - Morning briefing display
  - Task and schedule management UI

### APScheduler (Scheduler)
- **Purpose:** Manages time-based jobs: morning briefings, reminder delivery, weekly analytics regeneration, focus timer callbacks, schedule rerouting triggers
- **Technology:** Python APScheduler library
- **Scheduled jobs:**
  - Morning briefing (startup or configurable time)
  - Reminder firing (`reminders.trigger_at`)
  - Weekly analytics regeneration (Sunday night)
  - Focus timer check-ins (Pomodoro mid-point + end)

## Data Flow

### Voice Command → Action
```
Microphone
  → Wake word detection (Whisper / porcupine)
  → Speech transcription (Whisper STT)
  → Utterance normalisation (lowercase, punctuation strip)
  → Intent classification (NLP Parser, confidence ≥ 0.75)
  → Slot extraction (31 slot types)
  → Action Dispatcher
  → Handler (DB write / Macro Engine / Claude API call)
  → TTS response
  → voice_interactions log (SQLite)
```

### Session Lifecycle
```
Startup
  → Load last session from sessions table
  → Query pending session_pending_items
  → Generate morning briefing (Claude API)
  → Offer workspace setup (Macro Engine)

Shutdown
  → Snapshot active window + browser tabs
  → Prompt for shutdown notes (up to 3 exchanges)
  → Write session row + session_pending_items
  → Save DB snapshot
```

### Passive Monitoring → Analytics
```
Activity Monitor (30s)
  → activity_log INSERT
  → context_switches detection

Browser Extension (WebSocket)
  → browser_sessions INSERT

Weekly (Sunday night)
  → aggregate activity_log + tasks
  → INSERT analytics_weekly
  → Claude generates insight_summary

Morning
  → INSERT analytics_daily
  → Claude generates briefing_text
  → deliver_briefing intent served at startup
```

### Rubber Duck Mode
```
start_rubber_duck intent
  → Conversation mode: PROBLEM → CONSTRAINTS → EDGE_CASES → DEPS → CRITERIA → GENERATING
  → rubber_duck_sessions row built incrementally
  → generate_prd: Claude API call with full session transcript
  → PRD.md written to ~/Zeno/projects/<slug>/
  → Tasks extracted + added to tasks table
```

## Multi-Turn Dialogue State Machine

| Flow | Entry Intent | States | Exit Condition |
|------|--------------|--------|----------------|
| Day Planning | `start_day_planning` | LOADING → PRESENTING → BLOCKING → CONFIRMING → DONE | User confirms schedule or cancels |
| Rubber Duck | `start_rubber_duck` | PROBLEM → CONSTRAINTS → EDGE_CASES → DEPS → CRITERIA → GENERATING | Files written |
| Shutdown Ritual | `initiate_shutdown` | CAPTURING → PROMPTING → SAVING → DONE | User says "done" or 90s timeout |
| Re-routing | `rebalance_schedule` | ANALYSING → PROPOSING → AWAITING → APPLYING | All items resolved or "apply all" |
| Clarification | *(auto, confidence < 0.75)* | CLARIFYING → RESOLVED / ABANDONED | Max 2 rounds then abandon |

## Conventions

- **Naming:** Snake_case for Python modules and DB columns; kebab-case for workspace slugs and project slugs
- **Dates:** All dates stored as ISO 8601 strings (`YYYY-MM-DD`); times as `HH:MM` (24h); datetimes as SQLite `datetime('now')` format
- **Tags:** JSON arrays stored as TEXT columns (e.g., `'["bug","frontend"]'`)
- **Privacy:** All window title and browser domain writes checked against `privacy_exclusions` before INSERT
- **Security:** Macro Engine uses `app_classifications` whitelist — no arbitrary shell execution
- **Config duality:** Runtime config in `user_profile` (SQLite singleton) + `config.yaml` file (for human editing)
- **Testing:** Not yet established — no test files present in current artifact set
